## 🚀 服务器部署步骤

### 1. 连接到服务器

```bash
ssh your-server
```

### 2. 进入项目目录

```bash
cd /opt/weibo-ops
```

### 3. 拉取最新代码

```bash
git pull origin main
```

### 4. 安装依赖（如果有新依赖）

```bash
npm install
```

### 5. 构建项目

```bash
npm run build
```

### 6. 重启服务

```bash
# 如果使用 PM2
pm2 restart weibo-ops

# 或者如果使用 systemd
sudo systemctl restart weibo-ops
```

### 7. 验证部署

访问以下 URL 验证功能：
- `https://your-domain.com/performance` - 性能监控面板

### 浏览器控制台

## 📊 性能指标说明

| 指标 | 良好值 | 需要优化 |
|------|--------|----------|
| 平均响应时间 | < 500ms | > 1000ms |
| 缓存命中率 | > 80% | < 50% |
| 错误率 | < 1% | > 5% |

## ⚠️ 注意事项

1. **权限要求**
   - 只有 ADMIN 角色可以访问 `/performance` 页面
   - 其他角色访问会返回 404

2. **数据存储**
   - 性能数据存储在内存中
   - 服务重启后数据会清空
   - 建议定期查看和记录关键指标

3. **性能影响**
   - 监控系统本身开销很小（< 1ms）
   - 智能轮询在页面不可见时自动暂停
   - 不会影响生产环境性能

## 🐛 故障排查

### 问题：页面显示 404
**原因**：用户角色不是 ADMIN  
**解决**：确保使用 ADMIN 账号登录

### 问题：数据不更新
**原因**：轮询可能被暂停  
**解决**：
1. 检查页面是否可见
2. 打开浏览器控制台查看是否有错误
3. 刷新页面

### 问题：性能数据为空
**原因**：服务刚启动，还没有请求数据  
**解决**：等待几分钟，让系统收集数据
