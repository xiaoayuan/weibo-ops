# 数据隔离策略声明

> 建立时间：2026-05-04
> 更新时间：2026-05-05
> 状态：已确认（已与 owner 确认规则）

## 核心原则

本系统的数据隔离按以下策略划分——**按资源用途决定隔离级别，而非按用户角色**：

- **隔离型资源**：数据与具体用户账号绑定，包含敏感业务数据
- **公共型资源**：需要多用户协作维护，所有角色平等访问

---

## 隔离型资源（按用户账号隔离）

以下资源属于用户私有，**除 ADMIN 角色外，其他用户看不到也操作不了**：

| 资源 | 隔离规则 |
|------|----------|
| 微博账号 (`WeiboAccount`) | 账号归属于创建者的 `ownerUserId`，其他用户无法查看/编辑 |
| 每日计划 (`DailyPlan`) | 计划归属于账号，账号归属于用户，用户只能看到自己账号的计划 |
| 执行日志 (`ExecutionLog`) | 日志归属于账号，账号归属于用户，用户只能看到自己账号的执行日志 |
| 互动任务 (`InteractionTask`) | 任务归属于账号，用户只能看到自己账号的互动任务 |

### ADMIN 特权

- ADMIN 可以查看**所有用户**的以上隔离型资源，用于 bug 检测和全局运维
- ADMIN 查看执行日志时，**微博账号名统一替换为所属用户名**，防止信息泄露
  - 例如：`weibo_executor.ts` 中的账号 nickname 原为 "小明_微博"，ADMIN 看到的是 "zhangsan"（用户名）
  - 实现方式：查询后对 `account.nickname` 字段做替换，数据库实际不存储替换后的值
- ADMIN **不**自动拥有公共资源的特殊权限——公共资源的所有权属于"系统级"，所有角色平等访问

---

## 公共型资源（全局共享）

以下资源属于**公共资源**，**所有角色（ADMIN/OPERATOR/VIEWER）都可以查看、编辑和操作**：

| 资源 | 说明 |
|------|------|
| 文案库 (`CopywritingTemplate`) | 所有用户共同维护文案模板库，支持批量增删改 |
| 控评任务 (`ActionJob` / COMMENT_LIKE_BATCH) | 所有用户共同参与控评批次，共享评论池 |
| 转发轮转 (`ActionJob` / REPOST_ROTATION) | 所有用户共同参与转发轮转，共享轮转计划 |
| 超话配置 (`SuperTopic`) | 所有用户共同维护超话配置 |
| 话题任务 (`AccountTopicTask`) | 所有用户共同配置超话任务规则 |
| 代理节点 (`ProxyNode`) | 所有用户共享代理池 |
| 代理绑定 (`ProxyBinding`) | 所有用户共享代理分配策略 |
| 评论池 (`CommentLinkPoolItem`) | 所有用户共享控评评论链接池，支持批量导入 |

### 设计意图

这些公共资源的设计初衷是：**方便所有粉丝使用自己的账户进行同一种任务**。

- 粉丝 A 往文案库添加一条文案，粉丝 B 也可以使用
- 控评时所有人的账号共同参与，评论池共享
- 未来可能会改为用户私有，但**目前（2026-05-05）确定是公共的**

---

## 隔离型 vs 公共型 判断标准

按以下问题依次判断：

1. 这个资源的数据是否与具体用户账号绑定？ → **隔离型**
2. 这个资源是否需要多用户协作维护？ → **公共型**
3. 这个资源是否包含敏感的用户业务数据（如 cookie、密码）？ → **隔离型**
4. 这个资源是否支持批量协作（如批量文案、批量控评）？ → **公共型**

---

## 实现要点

### API 路由层

- 隔离型资源：查询时根据 `ownerUserId` 过滤，非 ADMIN 无法绕过
- 公共型资源：无 `ownerUserId` 过滤，所有人平等访问
- ADMIN 查看隔离型资源时调用遮蔽函数（`maskForeignAccounts`），将 `account.nickname` 替换为 `username`

### 前端展示层

- 隔离型资源页面：ADMIN 看到全部数据（带遮蔽），其他用户只看到自己的
- 公共型资源页面：所有用户看到相同数据，无账号选择器

### 权限检查

- 使用 `isAdmin(session)` 判断，跳过 `ownerUserId` 过滤
- 公共型资源操作只需 `OPERATOR` 角色（最低要求），ADMIN 无额外权限

---

## 已确认的例外规则

| 场景 | 规则 |
|------|------|
| ADMIN 查看执行日志 | 可见所有用户的日志，但微博账号名替换为用户名 |
| ADMIN 管理微博账号 | 可查看所有账号，但账号昵称以用户名显示 |
| 公共资源删除 | 任何 OPERATOR 都可以删除任意公共资源（无创建者限制） |
| 邀请码管理 | ADMIN 可以禁用任何邀请码（系统级资源，无用户归属） |

## 后续扩展

如果后续有新的资源类型，按以下问题判断归属：

- 这个资源的数据是否与具体用户账号绑定？ → 隔离型
- 这个资源是否需要多用户协作维护？ → 公共型
- 这个资源是否包含敏感的用户业务数据（如 cookie、密码）？ → 隔离型

如果要从公共型改为用户私有，需明确：
1. 旧数据如何迁移（公共→私有）
2. 现有任务的账号归属如何重新分配
3. 前端如何展示私有化后的资源
