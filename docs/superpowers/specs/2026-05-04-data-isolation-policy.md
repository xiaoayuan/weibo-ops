# 数据隔离策略声明

> 建立时间：2026-05-04
> 状态：已确认

## 核心原则

本系统的数据隔离按以下策略划分：

### 隔离型资源（按用户账号隔离）

以下资源属于用户私有，**除 ADMIN 角色外，其他用户看不到也操作不了**：

| 资源 | 隔离规则 |
|------|----------|
| 微博账号 (`WeiboAccount`) | 账号归属于创建者的 `ownerUserId`，其他用户无法查看/编辑 |
| 每日计划 (`DailyPlan`) | 计划归属于账号，账号归属于用户，用户只能看到自己账号的计划 |
| 执行日志 (`ExecutionLog`) | 日志归属于账号，账号归属于用户，用户只能看到自己账号的执行日志 |
| 互动任务 (`InteractionTask`) | 任务归属于账号，用户只能看到自己账号的互动任务 |

> **例外**：ADMIN 可以看见所有用户的以上资源，用于 bug 检测和全局运维。

### 公共型资源（全局共享）

以下资源属于**公共资源**，所有用户都可以查看、编辑和操作：

| 资源 | 说明 |
|------|------|
| 超话配置 (`SuperTopic`) | 所有用户共同维护超话配置 |
| 控评任务 (`ActionJob` / 控评轮转) | 所有用户共同参与控评批次 |
| 转发轮转 (`ActionJob` / 转发轮转) | 所有用户共同参与转发轮转 |
| 文案库 (`CopywritingTemplate`) | 所有用户共同维护文案模板库 |
| 话题任务 (`AccountTopicTask`) | 所有用户共同配置超话任务规则 |
| 代理节点 (`ProxyNode`) | 所有用户共享代理池 |
| 代理绑定 (`ProxyBinding`) | 所有用户共享代理分配策略 |

### ADMIN 特权

- 可以查看所有用户的微博账号
- 可以查看所有用户的执行日志
- 可以查看所有用户的每日计划
- 可以查看所有用户的互动任务
- 可以管理所有用户账号

ADMIN **不**自动拥有公共资源的特殊权限——公共资源的所有权属于"系统级"，所有角色都可以平等访问。

### 实现要点

1. **API 路由层**：查询时根据 `ownerUserId` 过滤，但 ADMIN 用户绕过此过滤
2. **前端展示层**：ADMIN 用户在账号/日志/计划等页面看到全部数据
3. **前端操作层**：公共资源页面不显示账号选择器（或默认显示全部可访问账号）
4. **权限检查函数**应统一为 `isAdmin(session) -> boolean`，供所有隔离型资源路由复用

### 后续扩展

如果后续有新的资源类型，按以下问题判断归属：
- 这个资源的数据是否与具体用户账号绑定？ → 隔离型
- 这个资源是否需要多用户协作维护？ → 公共型
- 这个资源是否包含敏感的用户业务数据（如 cookie、密码）？ → 隔离型
